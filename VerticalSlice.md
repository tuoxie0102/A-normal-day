# VerticalSlice.md · 灰盒 Demo 设计

> 目标：用纯 HTML/CSS/JS 构建最小可玩版本。
> 不用图片。不用动画。不用音效。只验证流程、节奏、交互。

---

## 一、范围

玩家完成 1 天完整循环：

```
TITLE → HOME → VENUE → OBSERVATION → DIALOGUE → RECORD → TITLE
```

仅 Day 1 · 陈思远。其他 3 人后续接入。

---

## 二、不需要的东西

| 不需要 | 替代方案 |
|--------|---------|
| 图片 | 纯色背景 + 文字描述 |
| 动画/过渡 | 直接替换 DOM |
| 音效 | 无 |
| 字体 | 系统默认中文字体 |
| 存档 | 无（每次从 TITLE 开始） |
| NPC 角色图 | 一个色块 + 名字标签 |
| 观察闪烁 | 虚线边框 + `[点击观察]` 提示 |
| 时间条 | 一个 `<progress>` 或文本 `剩余 12 分钟` |

---

## 三、UI 组件清单（灰盒版）

| 组件 | 实现 |
|------|------|
| 场景背景 | `<div>` + `background-color` |
| NPC 占位 | `<div>` 色块 + 名字 |
| 对话文本框 | `<div>` 深色底 + 白色文字 |
| 态度按钮 | 4 个 `<button>` + 1 个可选 `<button>` |
| 跳过按钮 | `<button>` 右上角 `跳过 ▸` |
| 冲突遮罩 | 全屏 `rgba(0,0,0,0.7)` |
| 关键句高亮 | `<div>` 深色底 + 左边框 |
| 第一印象面板 | 左侧 `<div>` 竖排标签 |
| 「他还没发现你」 | 2 个 `<button>` 居中 |
| 时间指示 | `剩余时间：12分钟` 文本 |
| 观察热点 | `<span>` 虚线边框 + 文字 |
| 记录面板 | `<div>` 结构化文本 |

---

## 四、状态流转（代码级）

```
state = TITLE
  → (click 开始游戏) → HOME
  → (auto 40s) → VENUE
  → (click 坐下/观察) → OBSERVATION 或 DIALOGUE_INTRO
  → (click 物品 或 时间耗尽 或 点坐下) → DIALOGUE_INTRO
  → (auto/skip) → DIALOGUE_CONFLICT1
  → (click 态度) → DIALOGUE_UNFOLD1
  → (auto/skip) → DIALOGUE_CONFLICT2
  → (click 态度) → DIALOGUE_UNFOLD2
  → (auto/skip) → DIALOGUE_JUDGMENT
  → (click 结局) → AFTERMATH
  → (click) → RECORD
  → (click 返回标题) → TITLE
```

```
demo.html          ← 单文件。包含 HTML + CSS + JS。
                   ← 引用 DEMO_NPC 数据（内嵌）。
                   ← 实现完整一天循环。
                   ← 不用任何外部依赖。
```

## 七、验证清单

Demo 完成后验证以下项目：

| 验证项 | 通过标准 |
|--------|---------|
| 一天流程走通 | TITLE → HOME → VENUE → DIALOGUE → RECORD → TITLE |
| 观察模式可用 | 点击物品 → 显示事实文本 → 标签出现 → 时间推进 |
| 跳过功能可用 | 自动段落中点击跳过 → 直接进入下一个互动节点 |
| 冲突节点可用 | 关键句高亮 → 4态度可选 → 选后看到 NPC 不同反应 |
| 最终判断可用 | 3结局可选 → 不同结局文字不同 |
| 记录生成正确 | 根据选择显示对应的第一印象、关键句、瑶珍记录 |
| 重玩结果不同 | 不同选择 → 不同展开段落 → 不同记录 |
| 无 JS 报错 | 控制台干净 |
| 在手机浏览器可运行 | 触屏点击正常。布局不自适应但可读。 |
