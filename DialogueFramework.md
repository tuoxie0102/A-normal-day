# DialogueFramework.md · 对话系统设计

> 不包含任何对白。只定义结构、节奏、交互规则。

---

## 一、一个 NPC 的对话由 6 个 Block 组成

```
Block 1：开场          自动 · 30s
Block 2：冲突节点 ①    互动 · 1 次选择
Block 3：展开 ①        自动 · 1min
Block 4：冲突节点 ②    互动 · 1 次选择
Block 5：展开 ②        自动 · 1min
Block 6：最终判断       互动 · 1 次选择
```

---

## 二、每个 Block 的定义

### Block 1 · 开场

| 属性 | 值 |
|------|-----|
| 类型 | 自动播放 |
| 时长 | ~30 秒（5-7 行） |
| 内容 | 场景确认 + 寒暄 + 第一印象建立 |
| 玩家操作 | 观看。可跳过。 |
| 结束时 | 自动进入 Block 2 |
| 为什么存在 | 让玩家进入情境。不需要互动——相亲开场天然是自动的。 |

### Block 2 · 冲突节点 ①

| 属性 | 值 |
|------|-----|
| 类型 | 互动 |
| 触发条件 | NPC 说出一句「关键句」（暴露他的第一个价值观矛盾） |
| 视觉变化 | 背景变暗。关键句高亮。态度选择按钮淡入。 |
| 玩家操作 | 从 4 个态度 + 可选 1 个语境选项中选 1 个 |
| AI 行为 | 根据态度 + 语境标签，生成一句瑶珍的回应 |
| NPC 反应 | 预设 3-5 句回应文本 |
| 结束时 | 自动进入 Block 3 |
| 为什么存在 | 第一次价值观碰撞。玩家的态度定义了后续对话的情绪走向。 |

### Block 3 · 展开 ①

| 属性 | 值 |
|------|-----|
| 类型 | 自动播放 |
| 时长 | ~1 分钟（3-5 行） |
| 内容 | 承接 Block 2 的情绪，对话自然推进到更深话题 |
| 路径 | 根据 Block 2 的选择分 2 条路径（理解/数据 为一组，追问/调侃 为另一组） |
| 玩家操作 | 观看。可跳过。 |
| 结束时 | 自动进入 Block 4 |
| 为什么存在 | 给玩家喘息。让 Block 2 的选择有时间「发酵」。 |

### Block 4 · 冲突节点 ②

| 属性 | 值 |
|------|-----|
| 类型 | 互动 |
| 触发条件 | NPC 说出一句「关键句」（暴露更深层矛盾——通常是情感核心） |
| 视觉变化 | 同 Block 2 |
| 玩家操作 | 同 Block 2 |
| 结束时 | 自动进入 Block 5 |
| 为什么存在 | 第二次价值观碰撞。比第一次更深——通常触及对方的恐惧或羞耻。 |

### Block 5 · 展开 ②

| 属性 | 值 |
|------|-----|
| 类型 | 自动播放 |
| 时长 | ~1 分钟 |
| 路径 | 根据 Block 4 的选择分 2 条路径 |
| 玩家操作 | 观看。可跳过。 |
| 结束时 | 自动进入 Block 6 |
| 为什么存在 | 收束情绪。为最终判断铺垫。 |

### Block 6 · 最终判断

| 属性 | 值 |
|------|-----|
| 类型 | 互动 |
| 触发条件 | NPC 说出一句暴露真实需求的话 |
| 玩家操作 | 从 3 个选项中选择 1 个 |
| 选项含义 | 🙂 接受 → 继续了解 / 🤔 保留 → 开放结局 / 😏 降级 → 转朋友 |
| 视觉变化 | 选择后播放道别动画（10-15s） |
| 结束时 | 触发 ACT 5（他去买单） |
| 为什么存在 | 玩家做出了今天最重要的决定。这个决定定义了本场相亲的结果。 |

---

## 三、自动 vs 互动分布

| Block | 类型 | 可跳过 |
|-------|------|--------|
| 开场 | 自动 | 是 |
| 冲突① | 互动 | 否 |
| 展开① | 自动 | 是 |
| 冲突② | 互动 | 否 |
| 展开② | 自动 | 是 |
| 最终判断 | 互动 | 否 |

**自动:互动 ≈ 70:30（时长比）**
**互动节点数量：3 个（不是 12 个，不是 20 个）**

---

## 四、交互密度数据

| 指标 | 数值 | 原因 |
|------|------|------|
| 一个 NPC 总时长 | 3-4 分钟 | 午休、通勤可完成 |
| 玩家点击次数 | 3 次（不含跳过） | 每次点击都改变走向 |
| 选择 → 可见后果 | 每次选择后 5-10 秒内 | 不让玩家等待反馈 |
| 跳过可用段落 | 3 段（开场 + 2 段展开） | 重玩时快速到达选择点 |
| 不可跳过段落 | 3 段（2 冲突 + 1 判断） | 这些是游戏，不是动画 |

---

## 五、设计原则

1. **不是每句话都值得回应。** 只有「关键句」才弹出选择。其余对白自动播放。

2. **每次选择都有后果。** 冲突①的选择改变展开①的走向。冲突②的选择改变展开②的走向。最终判断的选择决定结局。

3. **选择少但看得见。** 3 次选择，每次选完立刻看到 NPC 的不同反应。

4. **重玩友好。** 自动段落可跳过。3 次选择是唯一的路障。一个 NPC 重玩只需 ~1 分钟。
