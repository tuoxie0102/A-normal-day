# UIRevisionReport

## 本轮目标

本轮没有新增玩法、剧情或世界扩展。

只做一件事：

把 `prototype.html` 从“功能原型”收口成更像独立游戏 Vertical Slice 的可读片段。

优先级遵循：

- 玩家体验（UX）
- 阅读节奏
- 游戏氛围
- 美术细节
- 功能数量

## 修改页面

### 1. Breakfast Scene

- 保留爸爸左、妈妈右。
- 角色直接显示 PNG 原图，不再降低 Alpha。
- 角色整体放大，第一眼先看到人物。
- 对话框移到底部，减轻上半屏信息密度。
- 字体略缩小，留白增大，更接近 Coffee Talk 式阅读重心。

### 2. 世界观页

- 完全改成环境式表达。
- 使用整张城市广告图作为背景。
- 女主站在画面中央。
- 页面只保留“开始相亲”按钮。
- 不再用卡片和介绍文字替玩家解释世界。

### 3. 选择对象页

- 去掉资料卡结构。
- 只保留三个角色入口。
- 点击人物直接进入聊天。
- 页面的任务回到“选人”，而不是“读简介”。

### 4. Dialogue Scene

- 保留 Conversation Block，不恢复细碎的一句一句选择模式。
- 结构固定为：
- 环境提示
- NPC 自动聊天
- 玩家回应
- 下一 Block
- 对白框只显示说出口的话。
- 人物直接嵌入场景，不再独立成卡片。

### 5. 日记页

- 改成“晚上回家打开笔记本写下来的内容”。
- 使用暖色纸张、日期、天气、手写感排版。
- 去掉报告感、统计感、系统总结感。

## 动画方案

本轮只使用轻量 CSS 动画，没有新增复杂动画库。

### 已加入

1. Scene Fade

- 页面切换时整体淡入。

2. Character Slide

- 角色入场时轻微滑入。
- 位移控制在约 `30px`。
- 时长约 `420ms`。

3. Dialogue Fade

- 对白框淡入，不直接硬切。

4. Environment Hint

- 单句环境提示淡入、停留、淡出。
- 结束后再进入正式对白。

5. Button Hover

- 轻微缩放和颜色变化。

6. Camera Focus

- 对话阶段背景轻微压暗。
- 让玩家知道此刻需要看字、听人说话。

## 设计原则

### 1. UI 不抢戏

- 能交给场景表达的，不交给说明文字。
- 能交给人物站位表达的，不交给额外面板。

### 2. 阅读块尽量短

- 降低单屏信息量。
- 让玩家先看到人，再看到话。
- 让环境提示和对白分层。

### 3. 动画服务注意力，而不是展示技术

- 不做复杂镜头运动。
- 不做炫技转场。
- 只用微弱动效帮助视线落点。

## 关于外部方案与依赖

我做了轻量筛选，但本轮没有新增依赖。

评估过的方向：

- GSAP
- Open Props
- Motion One

最终选择：

- 本轮只用原生 CSS Transition / Keyframes

原因：

- 当前所需动画都很轻。
- 直接用 CSS 更容易维护。
- 不会让一个单文件 Vertical Slice 为了几十行过渡效果引入额外依赖。
- 更符合“动画只服务阅读和聊天体验”的目标。

## 本轮结论

这一轮的结果不是“更多功能”，而是更明确的体验重心：

- 更好读
- 更好听
- 更像在场景里相处
- 更有独立游戏的停顿感
