# Flow.md · 一天完整流程

> 设计目标：玩家在 5 分钟内完成一天。每一步的存在理由在下文说明。

---

## 流程图

```
TITLE SCREEN
    │
    ▼
ACT 1 · BREAKFAST（自动播放 · ~40s）
    │
    ▼
ACT 2 · ARRIVAL（自动播放 · ~5s）
    │
    ▼
ACT 3 · FIRST_CHOICE（1 次点击）
    │
    ├── SIT ──────────────┐
    │                      │
    └── OBSERVE ───┐       │
        │           │       │
        ▼           │       │
    OBSERVATION     │       │
    （点击探索）     │       │
    （最长 12min）   │       │
        │           │       │
        └───────────┴───────┘
                    │
                    ▼
ACT 4 · DIALOGUE（混合 · ~3-4min）
    Block 1: 开场（自动 30s）
    Block 2: 冲突①（互动）
    Block 3: 展开①（自动 1min）
    Block 4: 冲突②（互动）
    Block 5: 展开②（自动 1min）
    Block 6: 最终判断（互动）
                    │
                    ▼
ACT 5 · AFTERMATH（1 次可选点击 · ~15s）
                    │
                    ▼
ACT 6 · RECORD（自动 · ~10s）
                    │
                    ▼
NEXT DAY BUTTON
                    │
                    ▼
            返回 ACT 1（Day N+1）
```

## 每步说明

### ACT 1 · BREAKFAST

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 观看。不操作。 |
| 显示内容 | 家中餐桌。电视播放当天新闻。爸爸在厨房。妈妈在餐桌。3-5 句家庭对话。 |
| 为什么存在 | 建立情感锚点。每天从这里出发。新闻暗示当天相亲对象的社会处境。爸妈对话展示瑶珍为什么不讨好。 |
| 输出 | 场景过渡到 ACT 2。 |

### ACT 2 · ARRIVAL

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 观看。不操作。 |
| 显示内容 | 场景全景 2 秒 → 镜头推到相亲对象。对方处于自然状态。 |
| 为什么存在 | 场景切换缓冲。告知玩家地点、天气、对方第一印象。 |
| 输出 | 自动进入 ACT 3。 |

### ACT 3 · FIRST_CHOICE

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 点击 1 次。选项：「坐下开始聊天」或「先观察一下」。 |
| 为什么存在 | 核心分支。定义玩家获取信息的方式。选「坐下」→ 跳过观察，直接对话。选「观察」→ 进入观察模式。 |
| 输出 | 进入 ACT 4（对话）或 OBSERVATION 子状态。 |

### OBSERVATION 子状态

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 点击画面中闪烁的物品。每点 1 次推进 3 分钟。时间上限 12 分钟。超过后自动进入对话。 |
| 为什么存在 | 让玩家「读人」。观察结果产生「语境标签」，影响后续对话中的 AI 生成方向和可选选项。时间限制制造选择压力——不能全看。 |
| 输出 | 语境标签列表 → 传入 ACT 4。 |

### ACT 4 · DIALOGUE

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 自动段落观看（可跳过）。冲突节点从 4-5 个态度中选择 1 个。最终判断从 3 个结局方向中选择 1 个。 |
| 为什么存在 | 游戏的情感核心。玩家的选择定义了瑶珍的态度和关系走向。 |
| 输出 | 选择历史 + 结局方向 → 传入 ACT 5 和 ACT 6。 |

### ACT 5 · AFTERMATH

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 可选 1 次点击。「再看看桌上」或「等他回来」。 |
| 为什么存在 | 最后一次观察窗口。不影响结局，但影响《记录》的细节丰富度。 |
| 输出 | 1 条额外观察 → 传入 ACT 6。 |

### ACT 6 · RECORD

| 项目 | 说明 |
|------|------|
| 玩家做什么 | 阅读自动生成的《今日相亲记录》。不操作。 |
| 为什么存在 | 情绪收束。反馈——让玩家看到选择带来了什么。收藏——可回翻。 |
| 输出 | 记录保存到档案。显示「下一天」按钮。 |

## 一天时长预算

| 段落 | 时长 | 类型 |
|------|------|------|
| BREAKFAST | ~40s | 自动 |
| ARRIVAL | ~5s | 自动 |
| FIRST_CHOICE | ~2s | 互动 |
| OBSERVATION（可选） | 0~30s | 互动 |
| DIALOGUE | ~3-4min | 混合 |
| AFTERMATH | ~15s | 互动 |
| RECORD | ~10s | 自动 |
| **合计** | **~5min** | |
