# 第一部分：游戏最小循环（Core Loop）

```
┌──────────────────────────────────────────┐
│  第一幕：早晨（自动播放 · ~40秒）             │
│                                            │
│  玩家做什么：观看。                          │
│  为什么存在：建立世界观（新闻）、              │
│  展示瑶珍的家庭背景（爸妈对话）、               │
│  给出当天的情绪起点（便当细节）。               │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│  第二幕：进入场景（自动播放 · ~5秒）            │
│                                            │
│  玩家做什么：观看场景全景。                    │
│  为什么存在：场景切换缓冲。                    │
│  告知玩家地点、天气、氛围。                    │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│  第三幕：第一次选择（1次点击）                 │
│                                            │
│  玩家做什么：选择"坐下聊天"或"先观察"。         │
│  为什么存在：核心分支。                       │
│  - 选"坐下"→ 跳过观察，直接对话               │
│  - 选"观察"→ 进入观察模式                    │
│  这个选择定义了玩家获取信息的方式。             │
└──────────────────────────────────────────┘
                    ↓
          ┌────────┴────────┐
          ↓                 ↓
┌─────────────────┐  ┌─────────────────────┐
│  跳过观察         │  │  观察模式              │
│  直接进入第四幕    │  │  （点击探索 · 有时间限制） │
│                  │  │                      │
│  0次观察         │  │  玩家做什么：           │
│  0个语境标签     │  │  点击画面中闪烁的物品     │
│                  │  │  每点一次推进3分钟       │
│                  │  │  上限12分钟             │
│                  │  │                      │
│                  │  │  为什么存在：           │
│                  │  │  让玩家"读人"           │
│                  │  │  观察结果影响后续对话     │
│                  │  │  时间限制制造选择压力     │
└─────────────────┘  └─────────────────────┘
          │                 │
          └────────┬────────┘
                   ↓
┌──────────────────────────────────────────┐
│  第四幕：对话（核心游戏层 · ~3-4分钟）         │
│                                            │
│  结构：                                    │
│  开场(自动30s) → 冲突①(互动) → 展开(自动1min) │
│  → 冲突②(互动) → 展开(自动1min) → 最终判断(互动) │
│                                            │
│  玩家做什么：                               │
│  - 自动段落：观看（可跳过）                   │
│  - 冲突节点：从4-5个态度选项中选择1个          │
│  - 最终判断：从3个结局方向中选择1个            │
│                                            │
│  为什么存在：                               │
│  - 这是游戏的情感核心                        │
│  - 玩家的选择定义瑶珍的态度和关系走向          │
│  - 观察阶段收集的语境标签在此生效              │
└──────────────────────────────────────────┘
                   ↓
┌──────────────────────────────────────────┐
│  第五幕：他去买单（1次选择 · ~15秒）            │
│                                            │
│  玩家做什么：选择"再看看桌上"或"等他回来"。      │
│  为什么存在：最后一次观察机会。                 │
│  不影响结局，但影响《今日相亲记录》的细节。       │
└──────────────────────────────────────────┘
                   ↓
┌──────────────────────────────────────────┐
│  第六幕：记录生成（自动 · ~10秒）              │
│                                            │
│  玩家做什么：阅读生成的《今日相亲记录》。         │
│  为什么存在：                                │
│  - 情绪收束——让一天有完整的终点               │
│  - 收藏价值——玩家可以回翻                    │
│  - 反馈——让玩家看到自己的选择带来了什么         │
└──────────────────────────────────────────┘
                   ↓
┌──────────────────────────────────────────┐
│  过渡：点击"下一天"                          │
│  → 返回第一幕（Day N+1）                    │
└──────────────────────────────────────────┘
```

### 循环总结

| 步骤 | 玩家行为 | 时长 | 类型 |
|------|---------|------|------|
| 早晨 | 观看 | ~40s | 自动 |
| 进入场景 | 观看 | ~5s | 自动 |
| 第一次选择 | 1次点击 | ~2s | 互动 |
| 观察（可选） | 点击探索 | ~15-30s | 互动 |
| 对话 | 观看 + 3次选择 | ~3-4min | 混合 |
| 他去买单 | 1次点击 | ~2s | 互动 |
| 记录生成 | 阅读 | ~10s | 自动 |

**一天总时长：约 5 分钟。玩家互动次数：4-6 次（3 次冲突选择 + 1 次入场选择 + 0-1 次买单选择 + 若干观察点击）。**

# 第二部分：固定流程（每天不变）

以下内容每天固定出现，结构不随天数变化。

| 固定元素 | 内容 | 为什么固定 |
|---------|------|-----------|
| 早晨场景 | 家中早餐桌。电视播放新闻。爸爸在厨房。妈妈在餐桌。 | 建立稳定的情感锚点。玩家每次回来都从这个家出发。 |
| 新闻 | 电视/手机推送一条新闻标题 | 每天刷新世界观。暗示当天相亲对象的社会处境。 |
| 爸妈对话 | 3-5句。妈妈评价对象，爸爸用行动消解。 | 展示瑶珍的成长环境——为什么她不用讨好。也展示这个家庭内部已经存在的"另一种可能性"。 |
| 进入场景 | 场景全景 → 镜头推到相亲对象 | 场景切换缓冲。告知地点天气。 |
| "他还没发现你" | 两个选项：坐下聊天 / 先观察一下 | 每天的核心分支。固定出现才能形成习惯。 |
| 对话结构 | 开场 → 冲突① → 展开 → 冲突② → 展开 → 最终判断 | 固定的叙事节奏。玩家在第二天就知道"接下来会有一个冲突点"，可以预判。 |
| 态度选项 | 4个固定态度（🙂理解/🤔追问/😏调侃/📊数据）+ 可选🔍语境选项 | 一致的交互语言。玩家不需要每次重新学习操作。 |
| 他去买单 | 两个选项：再看看桌上 / 等他回来 | 最后一个观察窗口。固定的位置让玩家期待。 |
| 记录生成 | 《今日相亲记录》自动生成 | 每天都有完整收束。可回翻。 |
| "下一天" | 点击进入次日 | 过渡按钮。清晰的一天结束信号。 |

# 第四部分：对话结构

## 4.1 一个相亲对象的对话由 6 个 Conversation Block 组成

```
Block 1：开场
├── 类型：自动播放
├── 作用：建立场景、第一印象、自然寒暄
├── 时长：~30秒
├── 玩家操作：观看（可跳过）
└── 结束时：自动过渡到 Block 2

Block 2：冲突节点 ①
├── 类型：互动
├── 作用：暴露对方的第一个价值观矛盾
├── 触发：对方说出一句"关键句"
├── 视觉：背景变暗、关键句高亮
├── 玩家操作：从 4 个态度（+可选🔍）中选择 1 个
├── AI 行为：根据态度+语境标签生成瑶珍的回应
├── 反馈：对方做出反应（预设3-5句）
└── 结束时：自动过渡到 Block 3

Block 3：展开 ①
├── 类型：自动播放
├── 作用：承接冲突①的情绪，推进到更深话题
├── 时长：~1分钟
├── 路径：取决于 Block 2 的选择（2条主要路径）
├── 玩家操作：观看（可跳过）
└── 结束时：自动过渡到 Block 4

Block 4：冲突节点 ②
├── 类型：互动
├── 作用：暴露对方更深层的矛盾（通常是情感核心）
├── 触发：对方说出一句"关键句"
├── 视觉：同 Block 2
├── 玩家操作：从 4 个态度（+可选🔍）中选择 1 个
├── AI 行为：同 Block 2
├── 反馈：对方做出反应
└── 结束时：自动过渡到 Block 5

Block 5：展开 ②
├── 类型：自动播放
├── 作用：收束情绪，为最终判断铺垫
├── 时长：~1分钟
├── 路径：取决于 Block 4 的选择
├── 玩家操作：观看（可跳过）
└── 结束时：自动过渡到 Block 6

Block 6：最终判断
├── 类型：互动
├── 作用：决定本场相亲的结局方向
├── 触发：对方说出一句暴露真实需求的话
├── 玩家操作：从 3 个选项中选择 1 个
│   - 🙂 接受 → 继续了解
│   - 🤔 保留 → 开放结局
│   - 😏 降级 → 转朋友/队友/湖友/支持者
├── 反馈：播放道别动画
└── 结束时：触发《今日相亲记录》生成
```

## 4.2 自动 vs 互动分布

| Block | 类型 | 玩家操作 |
|-------|------|---------|
| 开场 | 自动 | 观看/跳过 |
| 冲突① | 互动 | 选择态度 |
| 展开① | 自动 | 观看/跳过 |
| 冲突② | 互动 | 选择态度 |
| 展开② | 自动 | 观看/跳过 |
| 最终判断 | 互动 | 选择结局 |

**自动段落总时长：约 2 分半。互动节点：3 个。**

## 4.3 跳过机制

- 自动段落中，右上角显示"跳过 ▸"
- 点击后：当前自动段落的文本一次性显示完毕，直接进入下一个互动节点
- 不能跳过互动节点（冲突节点和最终判断）
- 设计意图：尊重不耐烦的玩家。让重玩更高效。

# 第五部分：交互密度

## 5.1 数据

| 指标 | 数值 | 原因 |
|------|------|------|
| 单日总时长 | ~5分钟 | 一顿通勤、一次午休可以完成一天。不造成心理负担。 |
| 玩家点击次数 | 4-6次（不含观察点击） | 冲突①+冲突②+最终判断 = 3次必选。入场选择+买单选择 = 2次可选。再加上若干观察点击。 |
| 真正互动次数 | 3次（3个态度选择节点） | 这3次互动定义了一天的结果。其余是铺垫和收束。 |
| 自动对白比例 | ~70% | 大部分时间是"读"。互动只在关键转折点出现。 |
| 玩家选择比例 | ~30% | 选择少但每一次都影响走向。不做"选一个无关对话选项"的伪互动。 |

## 5.2 为什么这样设计

1. **不是每句话都值得回应。** 现实相亲中，真正改变关系走向的只有几句话。游戏的互动只放在这几句上。

2. **选择密度低但权重高。** 一天只有3次真正选择。玩家知道每次选择都重要，不会麻木地乱点。

3. **自动段落可跳过。** 让重玩（看不同结局）的成本降到最低。

4. **一周4天 ≈ 20分钟。** 一次完整通关在一段午休内可完成。

# 第六部分：未来扩展

## 6.1 扩展场景

| 男性数量 | 策略 |
|---------|------|
| 4个（当前） | 线性日历。每天一个。每个手写全部 Block。 |
| 20个 | 引入角色模板。每个角色 = 模板 + 2个冲突 + 6个观察点 + 社会期待层。模板不变，内容填充。 |
| 50个 | 部分角色共享对话模板（只换观察点和关键句）。引入随机匹配机制（每天从池中抽取1-3个可选对象）。 |
| 100个 | 引入程序化生成——每个角色的冲突从"冲突类型库"中组合（如：家庭压力型 ∩ 职业焦虑型 ∩ 自我物化型）。观察点从"观察点库"中按场景分配。 |

## 6.2 保证可扩展的设计原则

### 原则 1：模板不变，内容替换

```
对话模板（固定）：
  Block 1 开场     → 填充：角色A的30秒情境
  Block 2 冲突①   → 填充：角色A的核心矛盾1
  Block 3 展开①   → 填充：2条路径各1分钟
  Block 4 冲突②   → 填充：角色A的核心矛盾2
  Block 5 展开②   → 填充：2条路径各1分钟
  Block 6 最终判断 → 填充：角色A的3个结局
```

新增一个角色 = 填充6个Block的内容 + 6个观察点 + 对应场景。不修改模板。

### 原则 2：态度系统是通用接口

4个态度（🙂理解/🤔追问/😏调侃/📊数据）是所有角色的通用交互语言。新增角色时：
- 定义：该角色在冲突①和冲突②中，面对每个态度时的反应
- 定义：该角色的语境标签 → 第5选项映射
- 不需要修改态度系统的任何代码

### 原则 3：观察系统从"类型库"中分配

```
观察类型库（6类 × N条）：
  ① 行为类：[转杯子, 敲手指, 整理衣服, 频繁看手机, ...]
  ② 物品类：[手机, 书, 电脑包, 速写本, 文件夹, ...]
  ③ 外观类：[长袖, 文化衫, 卫衣, 毛衣, ...]
  ④ 环境类：[桌上的东西, 座位位置, 带的物品, ...]
  ⑤ 语言类：[说"应该", 说"我妈说", 说"效率", 说"对不起", ...]
  ⑥ 隐藏类：[连续点击触发, ...]
```

新增角色 = 从每个类型库中选1条，分配给他。

### 原则 4：社会期待层按场景模板填充

每个场景 = 3条固定环境文本 + 1条动态新闻 + 2条通用推送。新增场景时按模板填。

### 原则 5：日记模板按结局类型匹配

3种结局（继续/开放/朋友）× 每个角色的瑶珍语气 → 12个模板。新增角色 = 新增3条日记。

## 6.3 不需要重写的部分

| 永远不变 | 原因 |
|---------|------|
| 一天6幕流程 | 叙事节奏的基础骨架 |
| 对话Block结构（6个） | 最精简的有效对话单元 |
| 4种态度选项 | 通用交互语言 |
| 3种结局类型 | 关系走向的完整覆盖 |
| 《今日相亲记录》模板 | 结构化收藏品 |
| 早晨场景结构 | 情感锚点 |

# 附录：数据模板

## 新增一个角色需要提供的数据

```
角色ID：string
姓名：string
场景：string（对应assets/scenes/文件名）
天气：string + emoji
核心冲突标签：string

Block 1 · 开场：
  - 自动台词（5-7行，含说话人和叙述）

Block 2 · 冲突①：
  - 触发关键句（1句）
  - 态度 → 反应映射（4条：理解/追问/调侃/数据）

Block 3 · 展开①：
  - 路径A台词（取决于选了理解/数据，3-5行）
  - 路径B台词（取决于选了追问/调侃，3-5行）

Block 4 · 冲突②：
  - 触发关键句（1句）
  - 态度 → 反应映射（4条）

Block 5 · 展开②：
  - 路径A台词
  - 路径B台词

Block 6 · 最终判断：
  - 触发关键句（1句）
```
